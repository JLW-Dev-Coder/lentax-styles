Status Pill Color — Decision Aid Should the dashboard's functional status colors stay semantic, or follow the active theme?

The dashboard's functional status colors live in lentax-base.css §2.15 as --vlpd-success / --vlpd-warning / --vlpd-danger (currently fixed green / amber / red). They drive status pills and the reporting trend chips ("Past Due", completion %, "On Track", "Needs Attention"). Every other brand surface now re-skins per theme — these three are the last fixed colors. The question:

Option A — keep them semantic (success = green, warning = amber, danger = red, regardless of theme), or Option B — make them follow the theme accent (e.g. coastal teal/navy instead of green/red).

This is a decision aid, not a production file. It renders the pills two ways under each theme so the tradeoff is visible. Nothing here changes production CSS. Palette values are pulled verbatim from themes/tpp-default.css, themes/tpp-coastal.css, and lentax-base.css §2.15. A follow-up (Phase 5b) will implement whichever option JLW picks.

The tradeoff

OptionProsCons
A — Semantic
(fixed)		 Universal "red = bad / green = good" legibility. Status reads the same in every theme; no relearning. Accessibility-friendly (color meaning is stable). Green / amber / red can clash with a cool coastal (or other) surface — see the Coastal · Option A row above.
B — Theme
accent		 Fully cohesive look — pills always match the active brand palette. Loses the semantic color signal. Under Coastal, "Past Due" (navy) and "On Track" (teal) look alike — danger no longer reads as danger.
Decision for JLW: keep status colors semantic (A) or make them follow the theme (B)? Phase 5b will implement the pick — for (A), leave the three --vlpd-* functional tokens as literals; for (B), re-point them to themeable --lentax-* tokens.
